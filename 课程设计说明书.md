# AI手势交互游戏大厅 — 课程设计说明书

---

## 一、项目背景与需求分析

### 1.1 项目背景

随着计算机视觉和人机交互技术的快速发展，手势识别作为一种自然、非接触的交互方式，在游戏、教育、医疗等领域展现出巨大的应用潜力。传统的键盘鼠标操作需要使用者接触物理设备，而基于摄像头的手势识别可以让用户在三维空间中自由地控制游戏角色，带来更具沉浸感的游戏体验。

本项目旨在设计并实现一个基于手势识别的多游戏桌面应用。用户仅需使用电脑摄像头，无需任何穿戴设备，即可通过自然手势完成游戏选择、难度设置和实时游戏操作。系统集成了剪刀石头布、接水果、切水果等七款游戏，并通过大语言模型（LLM）实现动态关卡生成和虚拟对手人格化对话，充分展示了人工智能技术在游戏领域的创新应用。

### 1.2 需求分析

根据课程设计任务书要求，系统需实现以下功能：

1. **摄像头图像采集**：实时采集手势画面并预览
2. **手势识别**：采用OpenCV肤色检测或MediaPipe方案识别多种手势
3. **对战逻辑**：人机对战，支持多种游戏模式
4. **积分统计与数据持久化**：胜负判定、分数记录、对局存储
5. **语音播报**：对战结果语音反馈
6. **倒计时机制**：每局倒计时
7. **AI动态关卡生成**：调用LLM API根据玩家表现动态生成关卡数据
8. **AI虚拟对手人格化**：调用LLM API赋予对手角色人格，实时生成对话台词

### 1.3 项目定位

本项目为一款桌面级手势交互游戏平台，采用混合架构，Python负责摄像头采集与手势识别，Java负责游戏逻辑、UI渲染和数据管理。系统支持离线运行，LLM和语音播报作为增强功能，在网络不可用时自动降级为本地模式。

## 二、技术方案

### 2.1 总体技术架构

系统采用Python+Java混合架构，通过WebSocket实现进程间通信：

```
┌─────────────────────┐      WebSocket       ┌──────────────────────────┐
│  Python 程序         │ ───── JSON ──────→   │  Java 程序                │
│  MediaPipe 手势识别   │     localhost:8765   │  游戏大厅 + 7款游戏       │
│  摄像头 → 21个关键点   │                      │  JavaFX UI + Canvas渲染   │
│  手势分类 → JSON输出  │                      │  SQLite数据存储           │
└─────────────────────┘                      │  LLM API调用              │
                                             │  系统语音播报              │
                                             └──────────────────────────┘
```

### 2.2 技术选型

| 层面 | 技术 | 选型理由 |
|------|------|------|
| 手势识别 | Python + MediaPipe | 21个手部关键点3D坐标，准确率>95%，不受光线影响 |
| 通信方式 | WebSocket | 低延迟，双向通信，适合实时手势数据传输 |
| UI框架 | JavaFX + FXML | 成熟的Java桌面UI框架，支持CSS样式、FXML布局 |
| 游戏渲染 | JavaFX Canvas | 高效2D渲染，逐帧绘制游戏画面 |
| 数据存储 | SQLite | 轻量级嵌入式数据库，无需安装配置 |
| 语音播报 | edge-tts + SAPI | edge-tts提供自然语音，SAPI作为兜底 |
| LLM集成 | HTTP调用DeepSeek API | 异步调用，不阻塞游戏主循环 |
| 构建工具 | Maven | 依赖管理，一键打包 |

### 2.3 手势识别方案对比

| | OpenCV肤色检测 | MediaPipe Hand Landmarker |
|------|------|------|
| 原理 | HSV肤色过滤 → 轮廓提取 → 凸包凸缺陷 | 深度学习21点关键点检测 |
| 准确率 | ~80%（受光线影响） | ~99%（旋转无关角度判定） |
| 手势种类 | 3~4种 | 5种以上 |
| 开发难度 | 中（需调HSV阈值） | 低（预训练模型） |

本项目最终选择MediaPipe方案，以提供更准确、鲁棒的手势识别体验。在课程设计文档中保留开放CV方案作为备选方案的对比分析。

## 三、系统总体设计

### 3.1 系统模块划分

系统由八大模块组成：

1. **手势采集与识别模块（Python）**：负责摄像头画面采集和MediaPipe手势分类
2. **通信模块（Java WebSocket Server）**：接收Python端JSON数据，路由到各控制器
3. **大厅模块（LobbyController）**：游戏选择界面，手势滑动切换游戏，握拳确认
4. **游戏模块（GameInterface ×7）**：七款游戏的统一接口实现
5. **难度系统（Difficulty）**：简单/普通/困难/无尽四级难度
6. **UI渲染模块（GameRenderer）**：Canvas渲染、难度选择、结算画面
7. **数据持久化模块（SQLite）**：用户账户、最高分、对局记录
8. **增强模块（LLM + 语音）**：AI对手台词、塔罗牌解读、关卡生成、语音播报

### 3.2 系统状态机

```
STATE_AUTH（登录/注册）
    ↓ 登录成功
STATE_LOGIN（星空ENTER界面）
    ↓ 握拳确认
STATE_LOBBY（游戏大厅）
    ↓ 选游戏 → PEACE确认
STATE_DIFFICULTY（难度选择）
    ↓ 选难度 → FIST确认
STATE_GAME（游戏中）
    ↓ 游戏结束
STATE_GAME_OVER（结算）
    ↓ 握拳重玩 / 双手返回大厅
```

## 四、详细设计

### 4.1 手势识别模块

手势识别模块基于Google MediaPipe Hand Landmarker，通过分析手部21个关键点的3D坐标和关节角度实现旋转无关的手势分类。识别流程如下：

1. OpenCV打开摄像头，获取640×480 RGB帧
2. MediaPipe Hands处理，输出21个关键点归一化坐标
3. 计算关节角度（三个关键点间的夹角），替代传统的指尖Y轴比较
4. 根据手指伸开数量和组合判定手势类型

核心判定逻辑：

- **FIST（握拳）**：四根非拇指手指均蜷缩
- **OPEN（张开）**：四指以上伸开
- **PEACE（剪刀）**：食指+中指伸开，无名指小指蜷缩
- **POINTING（指向）**：仅食指伸开
- **THUMB_UP（点赞）**：拇指伸开，其他蜷缩

为提高面向不同手掌朝向的识别鲁棒性，手指伸开判定采用关节角度阈值（≥138°）和指尖-手腕距离比值（≥1.02），并通过PEACE和POINTING的多级兜底算法确保识别稳定性。

### 4.2 通信协议

Python端每帧通过WebSocket发送JSON数据到Java端的GestureStreamServer（端口8765）。报文格式：

```json
{
    "handX": 0.52, "handY": 0.61,
    "velocityX": 0.02, "velocityY": -0.02,
    "gesture": "fist", "confidence": 0.95,
    "handDetected": true, "handCount": 1,
    "navigationPalm": false,
    "image_data": "<base64_jpeg>"
}
```

有坐标均采用归一化坐标（0.0~1.0），左上角为原点，与JavaFX Canvas坐标系一致。

### 4.3 七款游戏设计

#### 4.3.1 接水果（CatchFruit）

玩家用手左右移动控制底部篮子，接住从上方掉落的水果并避开炸弹。难度影响篮子宽度、掉落速度、炸弹概率和生命数量。无尽模式下随着分数提高持续加速。

#### 4.3.2 祖玛（ZumaGame）

彩球链沿螺旋轨道持续向神庙入口推进。玩家无需把手移动到屏幕边缘，只需在摄像头中央舒适区小幅移动，系统便会将相对方向映射到整条外圈轨道，并在准星靠近彩球时提供轻量吸附。稳定握拳短暂保持后发射一颗彩球，握拳确认期间锁定枪口方向，避免手型变化引起发射偏移；发射后炮台自动装填另一种颜色。系统对握拳和松开分别进行连续帧确认并容忍短暂识别闪断，避免瞬时误判或持续握拳连发。多颗射弹可同时飞行，因此新的有效握拳不会因上一颗尚未落定而被忽略。射出的球插入球链后，连续三颗及以上同色球会被消除并产生连锁得分；彩球进入轨道终点则游戏结束。难度影响颜色数量、球链推进速度、目标分数和每波球数，无尽模式会持续生成新球链。

#### 4.3.3 剪刀石头布（RPSGame）

经典的猜拳游戏。玩家在摄像头前出手势（握拳=石头、张开=布、剪刀手=剪刀），与电脑AI对战。支持三局两胜、五局三胜、七局四胜三种模式。电脑AI根据难度调整策略：简单模式纯随机，困难模式会分析玩家出拳历史进行反制。每局判定后AI对手通过LLM生成个性化的嘲讽或鼓励台词，并通过语音播报。

#### 4.3.4 塔罗牌（TarotGame）

神秘占卜主题的卡牌游戏。3-7张牌阵排列在屏幕上，玩家移动手选择卡牌，握拳翻牌查看命运解读。牌面内容包含15种预设含义，翻牌动画采用宽度缩放模拟卡片翻转效果。张开手掌可重新洗牌。

#### 4.3.5 切水果（FruitNinja）

水果从屏幕底部沿抛物线弹出，玩家用手在空中划出刀光轨迹。通过点到线段距离算法检测轨迹与水果的碰撞。切中水果会生成果汁粒子效果并累积连击分数。炸弹碰触立即结束游戏。

#### 4.3.6 节奏大师（RhythmMaster）

跳舞机风格的音乐游戏。手势图标（✊✋✌）从屏幕底部向上滚动，到达判定圈时玩家需做出对应手势。判定时机分为Perfect（±3帧）、Great（±6帧）、Miss。Combo系统提供分数加成，60-120秒不等，结束时有银河主题评语。

#### 4.3.7 银河战机（SpaceShooter）

横向卷轴射击游戏。玩家用手控制战机上下移动，自动发射子弹消灭从右侧飞来的星际敌人。包含三种敌机类型（小兵、快速、厚血）和Boss战。支持道具系统：护盾、多弹道、回血、射速提升。通过LLM API动态生成敌机波次配置，实现AI驱动关卡生成。

### 4.4 难度系统

所有游戏统一采用四级难度：简单（EASY）、普通（NORMAL）、困难（HARD）、无尽（ENDLESS）。难度选择界面显示4张卡片，手移动选择，握拳确认。各游戏根据难度调整核心参数。

### 4.5 数据持久化

系统使用SQLite轻量级数据库存储两类数据：

1. **用户账户（users表）**：用户名、PBKDF2加盐密码摘要、创建时间
2. **最高分（high_scores表）**：游戏名称、难度、最高分数
3. **对局记录（match_history表）**：游戏名称、结果、分数、时间

### 4.6 LLM集成

系统通过HTTP异步调用DeepSeek（或兼容）API实现三处AI功能：

1. **猜拳对手台词**：每局判定后，LLM以嘲讽/鼓励口吻生成一句话（<20字），本地台词库兜底
2. **塔罗牌AI解读**：翻牌后，LLM以占卜师口吻生成2-3句个性化解读
3. **射击关卡生成**：每5波后，LLM根据玩家表现生成下一波敌机JSON配置

LLM不可用时，系统自动使用本地预设数据兜底。

### 4.7 语音播报

采用三级播报策略，确保跨电脑兼容性：

1. **edge-tts云健语音（优先）**：调用Microsoft Edge免费TTS，生成mp3播放，全电脑统一音色
2. **.NET Speech（备选）**：通过PowerShell调用系统自然语音
3. **SAPI VBS（兜底）**：最基础的Windows语音引擎

播报内容包括猜拳每局结果、对手台词、最终比分等。

### 5.2 手势识别测试

在不同光线条件和手掌朝向下测试手势识别准确率。MediaPipe方案在正常室内光线下准确率达到98%以上，背光条件下略有下降但仍保持在85%以上。通过添加角度判定和多级兜底算法，有效解决了传统Y轴比较方法在手掌侧对/背对摄像头时识别失败的问题。

### 5.3 性能测试

系统在Intel i5/16GB RAM/集成摄像头环境下，帧率稳定在60fps（通过AnimationTimer帧率锁）。WebSocket通信延迟<5ms，LLM异步调用不阻塞主循环。SQLite读写操作<10ms。

## 六、总结与展望

### 6.1 项目总结

本课程设计成功实现了一个基于手势识别的多功能游戏大厅，涵盖7款游戏、难度系统、LLM集成、语音播报和数据持久化。通过Python+Java混合架构，充分利用了MediaPipe的高精度手势识别和JavaFX的丰富UI能力。项目采用模块化设计，定义了统一的GameInterface接口，使得新增游戏只需实现8个方法即可无缝接入大厅。团队通过Git进行协作开发，使用AGENTS.md规范AI辅助编程的接口约定。

### 6.2 创新点

1. **混合架构**：Python处理AI模型，Java处理游戏逻辑，WebSocket桥接
2. **LLM集成**：将大语言模型应用于游戏台词生成和动态关卡设计
3. **跨平台语音**：edge-tts方案实现全电脑统一自然语音播报
4. **旋转无关手势识别**：基于关节角度的判定算法，不受手掌朝向影响

### 6.3 展望

未来可在以下方向继续完善：

1. 接入更多LLM API（如通义千问、文心一言）提供多样化AI体验
2. 增加用户等级和经验系统
3. 实现网络对战模式
4. 优化UI动画效果
5. 将视频帧动画背景替换为实时渲染粒子背景
